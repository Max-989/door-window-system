# 系统设计与迭代记录

## 文档目的

本文档记录门窗安装管理系统的系统架构、设计决策、更新历史和功能改进，用于持续迭代和知识传承。每次大的更新都会记录在此文档中，并同步更新到记忆系统。

## 更新原则

### 更新频率
- **大更新记录**：每次发布主要功能、架构变更、重大重构时更新
- **小更新汇总**：每周或每两周汇总小改进和bug修复
- **紧急修复**：生产环境紧急修复立即记录

### 更新标准（什么是"大的更新"）
1. **功能发布**：新增核心业务功能模块
2. **架构变更**：系统架构、技术栈、数据库结构重大调整
3. **性能优化**：显著提升系统性能的改进
4. **安全加固**：安全漏洞修复和加固措施
5. **用户体验**：主要界面重设计或交互流程重构
6. **技术债务**：重大技术债务清理和重构

### 记忆同步
每次大更新后，需要将更新摘要写入记忆系统，包括：
- 更新类型和范围
- 主要变更内容
- 影响评估
- 相关决策依据

## 系统架构概述

### 技术栈
- **后端**：Django 5.2 + Django REST Framework + MySQL
- **前端**：React 18 + TypeScript + Ant Design
- **移动端**：微信小程序
- **部署**：Docker + Docker Compose
- **CI/CD**：GitHub Actions

### 系统模块
1. **用户认证模块**：多角色用户体系、JWT认证、权限管理
2. **订单管理模块**：门窗订单全生命周期管理
3. **产品管理模块**：门窗产品库、规格配置
4. **仓库管理模块**：库存管理、出入库流程
5. **安装管理模块**：安装任务分配、进度跟踪
6. **维护管理模块**：售后维护、维修记录
7. **人员管理模块**：安装师傅、工头管理
8. **通知模块**：系统通知、消息推送
9. **报表模块**：业务数据统计和分析

### 数据架构
- **核心实体**：用户、订单、产品、库存、安装任务、维护记录
- **关系模型**：一对多、多对多关系支持业务复杂性
- **状态机**：订单状态、安装状态、维护状态等状态流转
- **审计跟踪**：关键操作日志记录

## 设计决策记录

### 2026-04-22：工程问题修复迭代
**决策**：采用Ralph自动化代理框架修复31个工程问题
**依据**：
1. 问题数量多且分散，人工修复效率低
2. Ralph支持最小迭代原则，每个故事可在单个上下文窗口完成
3. 遵循Andrej Karpathy编码原则：Think Before Coding, Simplicity First, Surgical Changes, Goal-Driven Execution
**影响**：
- 修复了API响应格式、认证安全、前后端枚举同步等关键问题
- 建立了自动化问题修复流程
- 更新了CI配置和.gitignore策略

### 2026-04-13：工程化实践改进
**决策**：全面实施工程化实践，从"手工作坊"升级为"工业化生产"
**依据**：
1. 项目规模扩大，需要标准化开发流程
2. 团队协作需要一致的代码质量和规范
3. 持续交付需要自动化测试和部署
**实施**：
- 配置代码质量工具（Black、Flake8、ESLint、Prettier）
- 建立CI/CD流水线（GitHub Actions）
- 完善Docker部署配置
- 添加健康检查端点

### 2026-04-09：前后端分离架构
**决策**：采用前后端分离架构，而非传统Django模板渲染
**依据**：
1. 需要支持Web管理后台和微信小程序多端
2. 前端技术栈现代化需求（React、TypeScript）
3. API优先设计便于第三方集成
**技术选型**：
- 后端：Django REST Framework提供RESTful API
- 前端：React + TypeScript + Ant Design
- 小程序：微信小程序原生开发

## 决策框架

### 技术选型原则
1. **成熟稳定**：优先选择经过生产验证的技术
2. **社区活跃**：确保有良好的社区支持和文档
3. **团队熟悉**：考虑团队技术栈和经验
4. **长期维护**：评估技术的生命周期和维护成本

### 架构演进原则
1. **渐进式演进**：避免大规模重写，小步快跑
2. **向后兼容**：确保API和数据的向后兼容性
3. **可观测性**：架构变更需包含监控和日志
4. **回滚预案**：重大变更必须有回滚方案

### 质量保障原则
1. **测试驱动**：重要功能必须有测试覆盖
2. **代码审查**：所有代码变更需要审查
3. **自动化检查**：利用CI/CD自动化质量检查
4. **文档同步**：代码变更与文档更新同步
