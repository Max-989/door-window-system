# 门窗安装管理系统 - 数据库设计文档

## 版本信息
- **版本**: 1.1
- **日期**: 2026-04-06
- **基于**: 实施计划阶段1需求 + 现有`.base`文件分析

## 设计原则
1. **业务驱动设计**: 基于门窗安装全生命周期业务流程
2. **数据迁移友好**: 兼容现有`.base`文件数据结构
3. **扩展性**: 支持多装企、多服务人员、多产品类型
4. **权限控制**: 细粒度角色权限和数据隔离

## 实体关系图概览

```
用户体系 (User/Role/Permission)
    ├── 客户体系 (Customer/CompanyCustomer/DirectCustomer)
    ├── 服务人员体系 (ServiceProvider/Factory/ServiceContractor/ServicePerson)
    └── 业务体系
        ├── 订单生命周期 (Order/OrderItem)
        ├── 量尺流程 (MeasurementRequest/MeasurementTask)
        ├── 安装流程 (InstallationTask)
        ├── 维修流程 (MaintenanceRequest)
        ├── 产品库存 (Product/Material/Inventory)
        └── 辅助实体 (Location/Payment/Notification)
```

## 实体详细设计

### 1. 用户与权限实体

#### 1.1 User (用户)
- **对应现有表**: 可能为新表
- **用途**: 系统所有用户的基类，支持微信/手机号登录
- **字段**:
  - `id`: 主键
  - `phone`: 手机号 (唯一)
  - `email`: 邮箱 (可选)
  - `wechatOpenId`: 微信OpenID (唯一，可选)
  - `wechatUnionId`: 微信UnionID (可选)
  - `password`: 加密密码 (可选，微信用户无密码)
  - `realName`: 真实姓名
  - `idCard`: 身份证号
  - `avatar`: 头像URL
  - `role`: 用户角色 (enum: admin=系统管理员, clerk=文员, warehouse=仓库人员, project_manager=项目经理, sales=导购, measurer=量尺师傅, installer=安装师傅, repairman=维修师傅, foreman=工头)
  - `status`: 用户状态 (enum: active, inactive, suspended)
  - `lastLoginAt`: 最后登录时间
  - `loginCount`: 登录次数
  - `createdAt`, `updatedAt`: 时间戳

#### 1.2 Role (角色)
- **对应现有表**: 可能为新表
- **用途**: 定义角色模板
- **字段**:
  - `id`: 主键
  - `name`: 角色名称 (唯一)
  - `code`: 角色代码 (唯一，如: admin, factory_manager, service_manager等)
  - `description`: 角色描述
  - `permissions`: 权限JSON配置
  - `createdAt`, `updatedAt`: 时间戳

#### 1.3 Permission (权限)
- **对应现有表**: 可能为新表
- **用途**: 权限细粒度控制
- **字段**:
  - `id`: 主键
  - `code`: 权限代码 (唯一)
  - `name`: 权限名称
  - `description`: 权限描述
  - `resource`: 资源类型 (order, measurement, installation等)
  - `action`: 操作类型 (create, read, update, delete, approve等)
  - `createdAt`, `updatedAt`: 时间戳

### 2. 客户实体

#### 2.1 Customer (客户基类)
- **对应现有表**: 可能为"客户"表
- **用途**: 所有客户的基类
- **字段**:
  - `id`: 主键
  - `code`: 客户编码 (唯一)
  - `name`: 客户名称
  - `type`: 客户类型 (enum: company, direct)
  - `contactPerson`: 联系人
  - `phone`: 联系电话
  - `email`: 邮箱
  - `address`: 地址
  - `region`: 所在区域
  - `status`: 客户状态 (enum: active, inactive)
  - `totalOrders`: 总订单数
  - `totalAmount`: 总交易金额
  - `createdAt`, `updatedAt`: 时间戳

#### 2.2 CompanyCustomer (装企客户)
- **继承自**: Customer
- **对应现有表**: "装企目录" + 客户信息
- **用途**: 装修企业客户
- **字段**:
  - `customerId`: 关联Customer主键
  - `companyType`: 装企类型 (enum: design_company, construction_company等)
  - `cooperationLevel`: 合作级别
  - `commissionRate`: 佣金率
  - `contractStartDate`: 合同开始日期
  - `contractEndDate`: 合同结束日期
  - `salesPersonId`: 专属销售员ID

#### 2.3 DirectCustomer (直接客户)
- **继承自**: Customer
- **用途**: 直接终端客户
- **字段**:
  - `customerId`: 关联Customer主键
  - `source`: 客户来源 (enum: recommendation, advertisement, website等)
  - `isVip`: 是否VIP客户
  - `vipLevel`: VIP等级

### 3. 服务人员实体

#### 3.1 ServiceProvider (服务提供者基类)
- **用途**: 所有服务提供者的基类
- **字段**:
  - `id`: 主键
  - `code`: 服务商编码 (唯一)
  - `name`: 服务商名称
  - `type`: 服务商类型 (enum: factory, contractor, person)
  - `contactPerson`: 联系人
  - `phone`: 联系电话
  - `email`: 邮箱
  - `address`: 地址
  - `region`: 服务区域
  - `status`: 状态 (enum: active, inactive)
  - `serviceTypes`: 服务类型数组 (measurement, installation, maintenance等)
  - `createdAt`, `updatedAt`: 时间戳

#### 3.2 Factory (厂家)
- **继承自**: ServiceProvider
- **对应现有表**: 可能为"厂家"相关表
- **用途**: 门窗生产厂家
- **字段**:
  - `serviceProviderId`: 关联ServiceProvider主键
  - `factoryType`: 厂家类型 (enum: window, door, accessory等)
  - `productionCapacity`: 生产能力
  - `qualityRating`: 质量评级
  - `deliveryTime`: 平均交货时间(天)

#### 3.3 ServiceContractor (服务承接商)
- **继承自**: ServiceProvider
- **对应现有表**: 可能为"服务商"相关表
- **用途**: 安装服务承接商
- **字段**:
  - `serviceProviderId`: 关联ServiceProvider主键
  - `contractorType`: 承接商类型 (enum: measurement, installation, maintenance)
  - `workerCount`: 工人数量
  - `serviceRating`: 服务评级
  - `contractTerms`: 合同条款

#### 3.4 ServicePerson (服务人员)
- **继承自**: ServiceProvider
- **对应现有表**: "安装师傅表"
- **用途**: 具体的服务人员(量尺师、安装师傅等)
- **字段**:
  - `serviceProviderId`: 关联ServiceProvider主键
  - `personType`: 人员类型 (enum: manager, measurement_specialist, installer, after_sales)
  - `userId`: 关联User ID (可选，如果是系统用户)
  - `idCard`: 身份证号
  - `skillLevel`: 技能等级
  - `workStatus`: 工作状态 (enum: available, busy, offline)
  - `averageRating`: 平均评分
  - `completedTasks`: 完成任务数

### 4. 业务实体

#### 4.1 Order (销售订单)
- **对应现有表**: "销售订单"、"订单"
- **用途**: 销售订单主表，全生命周期管理
- **字段**:
  - `id`: 主键
  - `orderNo`: 订单编号 (唯一，规则: CP-XXXXXXXX)
  - `customerId`: 客户ID
  - `customerType`: 客户类型 (company, direct)
  - `companyCustomerId`: 装企客户ID (如果是装企客户)
  - `salesPersonId`: 销售员ID
  - `designerId`: 设计师ID
  - `measurementRequestId`: 关联的量尺需求ID
  - `totalAmount`: 订单总金额
  - `paidAmount`: 已付金额
  - `discountAmount`: 折扣金额
  - `status`: 订单状态 (enum: draft, submitted, reviewing, confirmed, cancelled)
  - `productionStatus`: 生产状态 (enum: pending, in_production, completed, shipped_to_warehouse, arrived_warehouse, shipped_to_site, arrived_site)
  - `installationStatus`: 安装状态 (enum: pending_assignment, assigned, scheduling, in_progress, completed, accepted, rated)
  - `paymentStatus`: 支付状态 (enum: pending, partial, completed, refunded)
  - `orderDate`: 订单日期
  - `expectedDeliveryDate`: 预计交货日期
  - `actualDeliveryDate`: 实际交货日期
  - `installationAddress`: 安装地址
  - `notes`: 备注
  - `createdAt`, `updatedAt`: 时间戳

#### 4.2 OrderItem (订单明细)
- **对应现有表**: "订单明细"
- **用途**: 订单产品明细
- **字段**:
  - `id`: 主键
  - `orderId`: 订单ID
  - `productId`: 产品ID
  - `productName`: 产品名称
  - `productSpec`: 产品规格
  - `quantity`: 数量
  - `unitPrice`: 单价
  - `totalPrice`: 总价
  - `materialCost`: 材料成本
  - `laborCost`: 人工成本
  - `notes`: 备注
  - `createdAt`, `updatedAt`: 时间戳

#### 4.3 MeasurementRequest (量尺需求)
- **对应现有表**: "已量尺检测"相关
- **用途**: 量尺需求申请
- **字段**:
  - `id`: 主键
  - `requestNo`: 需求编号 (唯一)
  - `customerId`: 客户ID
  - `customerType`: 客户类型
  - `requestedBy`: 申请人ID (项目经理/设计师/销售员/客户)
  - `requestType`: 申请类型 (enum: new, followup, repair)
  - `propertyAddress`: 物业地址
  - `propertyType`: 物业类型 (enum: residential, commercial, villa, apartment)
  - `measurementDate`: 期望量尺日期
  - `timeSlot`: 时间段
  - `specialRequirements`: 特殊要求
  - `status`: 需求状态 (enum: pending, submitted, assigned, in_progress, completed, reviewed)
  - `priority`: 优先级 (enum: low, medium, high, urgent)
  - `createdAt`, `updatedAt`: 时间戳

#### 4.4 MeasurementTask (量尺任务)
- **对应现有表**: 可能为"量尺任务"相关
- **用途**: 量尺任务分配和执行
- **字段**:
  - `id`: 主键
  - `measurementRequestId`: 量尺需求ID
  - `assignedTo`: 分配给 (量尺师傅ID)
  - `scheduledDate`: 安排日期

  - `notes`: 量尺备注
  - `status`: 任务状态 (enum: assigned, confirmed, in_progress, completed, cancelled)

  - `createdAt`, `updatedAt`: 时间戳

#### 4.5 InstallationTask (安装任务)

  - `assignedTo`: 安装师傅ID

#### 4.6 MaintenanceRequest (维修申请)

## 数据迁移策略

### 迁移步骤
1. **分析阶段**: 使用现有Python脚本分析`.base`文件结构
2. **映射设计**: 建立旧表字段到新实体的映射关系
3. **数据清洗**: 清理无效数据，标准化格式
4. **分批导入**: 按依赖关系分批导入数据
5. **验证检查**: 数据完整性验证

### 迁移优先级
1. **基础数据**: User, Customer, DecorateCompany, Product
2. **业务数据**: Order, OrderItem
3. **流程数据**: MeasurementRequest, InstallationTask, MaintenanceRequest
4. **交易数据**: Payment
5. **辅助数据**: Location, Notification
