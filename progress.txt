## Codebase Patterns
- request.ts auto-unwraps unified responses: if response JSON has `code` and `data` fields, returns `json.data` directly. Callers get the data payload, not the wrapper.
- After auto-unwrap, use `Array.isArray(res) ? res : (res.items || res.results || [])` for list data and `res.total ?? res.count ?? 0` for total count.
- Never access `.data` on request responses — it's already unwrapped.
- Django project root is `backend/`. Config files are in `backend/config/settings/`.
- Backend tests are run with `pytest` using `config.settings.test` settings. Requires `pytest-django` installed.
- Linting uses `flake8` with configuration in `.flake8`.
- Frontend type checking uses `tsc --noEmit` via `npm run type-check`.
- Secret key environment variable is `DJANGO_SECRET_KEY` (changed from `SECRET_KEY`).
- Base config sets `DEBUG = False`, dev overrides to `True`.
- When overriding nested settings like REST_FRAMEWORK, use copy.deepcopy to preserve base configuration.
- Permission store empty permissions/menus should deny access (security default).
- To avoid import conflicts between `common/utils.py` and `common/utils/` directory, rename file to `utils_core.py` and create `__init__.py` that re-exports symbols.
- Use common.responses.success/error/created for unified API responses.
- Use `common.pagination.StandardPagination` for paginated list APIs; set `pagination_class` attribute on `ListAPIView` or `GenericViewSet`.
- For token refresh, deduplicate concurrent 401 requests via shared promise to avoid multiple refresh calls.
- Auth store must persist `refresh` token alongside `access` token in zustand partialize.
- Token refresh endpoint: use SimpleJWT `TokenRefreshView.as_view()` wrapped in unified response format.
- Frontend login must unwrap `response.data` after unified API response change (US-004).
- Use `GenericViewSet` (not `ViewSet`) when view needs pagination. `ViewSet` lacks `paginate_queryset` and `get_paginated_response`.
- When using `GenericViewSet` with custom `list` action, manually call `self.paginate_queryset()` and `self.get_paginated_response()`.
- For frontend API calls, use `request` utility (`get`/`post`) instead of raw `fetch()` to get auth headers and token refresh.
- StandardPagination response format: `{code, message, data: {items, total, page, pageSize, totalPages}, timestamp}`.
- Linting uses `flake8` with configuration in `.flake8`.
- Frontend type checking uses `tsc --noEmit` via `npm run type-check`.
- Secret key environment variable is `DJANGO_SECRET_KEY` (changed from `SECRET_KEY`).
- Base config sets `DEBUG = False`, dev overrides to `True`.
- When overriding nested settings like REST_FRAMEWORK, use copy.deepcopy to preserve base configuration.
- Permission store empty permissions/menus should deny access (security default).
- To avoid import conflicts between `common/utils.py` and `common/utils/` directory, rename file to `utils_core.py` and create `__init__.py` that re-exports symbols.
- Use common.responses.success/error/created for unified API responses.
- Use `common.pagination.StandardPagination` for paginated list APIs; set `pagination_class` attribute on `ListAPIView`.
- For token refresh, deduplicate concurrent 401 requests via shared promise to avoid multiple refresh calls.
- Auth store must persist `refresh` token alongside `access` token in zustand partialize.
- Token refresh endpoint: use SimpleJWT `TokenRefreshView.as_view()` wrapped in unified response format.
- Frontend login must unwrap `response.data` after unified API response change (US-004).
