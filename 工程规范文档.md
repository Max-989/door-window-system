```json
  "code": 200,

  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "pageSize": 20,
    "totalPages": 5
```

```json
{
```

### 1.4 分页规范

- **默认分页器**：`common.pagination.StandardPagination`
- **默认每页条数**：20
- **分页参数**：`page`（页码，从1开始）、`pageSize`（每页条数，最大100）
- **前端自动解包**：`request.ts` 直接返回 `data` 层，调用方拿 `{ items, total }`

### 1.5 认证规范

- **认证方式**：JWT Bearer Token（SimpleJWT）
- **Token 有效期**：Access Token 15分钟，Refresh Token 7天
- **Token 刷新**：支持 rotate refresh token
- **请求头**：`Authorization: Bearer <token>`
- **Token 存储**：前端 `localStorage` key=`auth-storage`（zustand persist）

### 1.6 错误处理

- **全局异常处理器**：`common.exceptions.custom_exception_handler`
- **HTTP 状态码与业务 code 保持一致**
- **字段验证错误**：`errors` 数组，包含 `field` 和 `message`
- **401 未认证**：前端自动跳转登录页
- **前端错误处理**：`request.ts` 抛出 `{ status, data }` 错误对象

### 1.7 筛选与搜索

- **筛选参数**：直接使用字段名，如 `?status=pending&brand=1`
- **搜索**：`?search=关键词`（匹配 `search_fields` 定义的字段）
- **排序**：`?ordering=-created_at`（`-` 降序）
- **时间范围**：`?date_start=2026-01-01&date_end=2026-12-31`
- **默认排序**：所有模型按 `-created_at` 降序

### 2.3 ViewSet 规范

- **标准 CRUD**：继承 `viewsets.ModelViewSet`
- **权限**：统一使用 `IsAuthenticated`，管理员操作使用 `IsAdminUser`
- **过滤**：`DjangoFilterBackend + SearchFilter + OrderingFilter`
- **序列化器选择**：通过 `get_serializer_class()` 方法区分 list/detail/create/update
- **自定义 action**：使用 `@action(detail=True/False, methods=[...])`

### 2.4 序列化器规范

- `fields = "__all__"` 用于完整序列化
- `read_only_fields = ["created_at", "updated_at"]`
- 关联字段用 `source="field.name"` 添加 `display` 字段
- 分离搜索序列化器（如 `WoodProductSearchSerializer`）减少数据传输

### 2.5 枚举规范

- **集中定义**：`common/enums.py`，使用类常量模式
- **格式**：`CHOICES = [(VALUE, LABEL), ...]`
- **模型字段**：`models.CharField(max_length=20, choices=EnumClass.CHOICES)`
- **命名约定**：大驼峰类名，全小写下划线值，中文 label

### 2.6 模型规范

- **表名**：`db_table` 显式指定，复数形式（如 `orders`, `users`）
- **时间戳**：`created_at`（auto_now_add）+ `updated_at`（auto_now）
- **排序**：默认 `ordering = ["-created_at"]`
- **外键**：`on_delete=SET_NULL`（大部分场景），`on_delete=CASCADE`（强关联）
- **JSON 字段**：`models.JSONField(default=list/dict, blank=True)`
- **金额**：`models.DecimalField(max_digits=12, decimal_places=2)`

### 3.3 路由规范

- **公开路由**：`/login`, `/register`
- **受保护路由**：`/app/*`（需要登录）
- **权限路由**：通过 `PermissionRoute` 组件检查菜单权限
- **路由守卫**：`ProtectedRoute`（认证检查）→ `PermissionRoute`（权限检查）
- **未授权回退**：跳转到 `/403` 或 `/login`

### 3.4 状态管理规范

- **store 定义**：使用 `create<T>()(persist(...))` 模式
- **持久化**：`localStorage` key=`auth-storage`，`partialize` 指定持久化字段
- **API 调用**：store 内部直接调用 `request.ts` 方法
- **权限放行策略**：API 失败或权限为空时默认放行，避免白屏

### 3.5 HTTP 请求规范

- **BASE_URL**：`/api/v1`（通过 vite proxy 转发到 localhost:8000）
- **Token 获取**：从 `localStorage` 的 `auth-storage` 中读取
- **401 处理**：自动清除登录状态并跳转登录页
- **导出方法**：`get`, `post`, `put`, `del`, `patch`
- **FormData 支持**：自动检测 `body instanceof FormData`，不设 Content-Type

### 3.6 TypeScript 规范

- **strict 模式**：全开（`noImplicitAny`, `strictNullChecks` 等）
- **路径别名**：`@/*` → `src/*`
- **类型定义**：集中到 `types/api.ts`
- **接口命名**：PascalCase

### 3.7 公共组件规范

- **BaseTable**：服务端分页（`pagination` prop）+ 客户端分页自动切换
- **Layout**：侧边栏菜单 + 面包屑 + 用户信息
- **StatusTag**：根据状态值映射颜色和文本
- **ModalForm**：通用弹窗表单（新建/编辑复用）

## 四、数据库规范

### 4.1 命名规范

- **表名**：蛇形复数（`orders`, `users`, `wood_products`）
- **字段名**：蛇形（`customer_name`, `order_no`）
- **主键**：默认 BigAutoField（`id`）
- **外键字段**：`xxx_id` 命名

### 4.2 字段规范

| 字段类型 | Django 类型 | 说明 |
|---------|------------|------|
| 字符串 | `CharField(max_length=N)` | 必须指定 max_length |
| 文本 | `TextField(blank=True, default="")` | 长文本 |
| 金额 | `DecimalField(max_digits=12, decimal_places=2)` | 所有价格/费用 |
| 枚举 | `CharField(choices=...)` | max_length=20 |
| JSON | `JSONField(default=list/dict)` | 数组或对象 |
| 时间 | `DateTimeField(auto_now_add/auto_now)` | 自动维护 |
| 布尔 | `BooleanField(default=...)` | - |
| 图片 | `ImageField(upload_to=...)` | - |

### 4.3 索引规范

- **唯一索引**：`unique=True`（如 `order_no`, `task_no`, `phone`）
- **普通索引**：`db_index=True`（如 `customer_address` 的模糊查询字段）
- **外键**：自动创建索引（Django 默认行为）
- **联合索引**：`unique_together`（如 `perm_role_members`）

### 4.4 约束规范

- `blank=True, default=""` 用于可选字符字段
- `null=True, blank=True` 用于可选外键
- `on_delete=SET_NULL` 用于弱关联（品牌、门店等可删除）
- `on_delete=CASCADE` 用于强关联（订单明细、结算记录）

## 八、安全规范

### 8.1 认证安全

- JWT Token 15分钟过期，自动刷新
- Refresh Token 轮换 + 黑名单
- 密码哈希：Django PBKDF2

### 8.2 CORS

- 开发环境：允许 `localhost:3000-3002` 和 `127.0.0.1:3000-3002`
- 生产环境：需配置 `CORS_ALLOWED_ORIGINS`

### 8.3 API 限流

- 匿名用户：100 次/小时
- 认证用户：1000 次/小时

### 8.4 敏感信息

- 数据库密码、Secret Key 等通过环境变量注入
- 生产环境 `DEBUG = False`
- `ALLOWED_HOSTS` 需显式配置
